✅useSWRはurl指定してfetchして使用するので、URLを持たないサーバーアクションは使用できない


✅なぜサーバーアクションはurlを持たないのか
urlを持つファイルは以下
page.tsx
route.ts

★ではサーバーアクションをpage.tsxにかけばよいか
結論駄目！
page.tsx は ブラウザに表示される UI コンポーネントであり、
Server Action を API の代わりとして公開する用途ではない。


★route.tsにサーバーアクション書いてfetchしたとしても
サーバーアクションは命名関数なのでimport対象として呼び出さないと実行できないので
fetchしても無意味

✅ルートハンドラーズの場合はfetchすれば指定したHTTPメソッドに記述した関数は実行されるのでバッチリOK
