✅google認証は特定のgoogleアカウントを使って設定を行い そのアカウントでクライアントIDやクラインとシークレットを発行してsupabaseとひもずけるが、 特定のアカウントで設定は行うが、ログイン自体はどのアカウントでも出来るようになる。


1. 設定用のGoogleアカウント
Google API ConsoleでOAuthクライアントIDを作るために、1つのGoogleアカウントで設定を行います。
このアカウントは「設定者」として使われるだけで、ログインの対象ではありません。
設定時にクライアントIDとクライアントシークレットが発行されます。

2. Supabaseとの紐付け
発行したクライアントID・シークレットをSupabaseの「Auth → Providers → Google」に設定します。
これでSupabase側がGoogleログインの処理を理解できるようになります。

3. 実際のログイン
この設定は Google OAuthアプリケーション全体に対する設定 なので、他のGoogleアカウントでもログイン可能です。
つまり、自分が設定したGoogleアカウント以外のメールでもOAuthログインできます。

✅ ポイント
「設定するGoogleアカウント」と「ログインできるGoogleアカウント」は別物。
単にgoogle認証を使えるようにする為に、ある特定のアカウントでクライアントIDやクラインとシークレットを発行したりして
設定する必要があるというだけ。
